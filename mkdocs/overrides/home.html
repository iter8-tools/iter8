Declarative Experiments

Iter8 experiments enable DevOps/MLOps/SRE teams to easily implement SLO validation, A/B(/n) testing with business metrics, chaos injection, dark launch, canaries, progressive rollouts with advanced traffic engineering, and their hybrids.

Built-in/Custom Metrics

Iter8's built-in metrics collection feature can be used to generate synthetic HTTP GET/POST requests and collect latency and error related metrics for apps. If you collect metrics metrics for your app in Prometheus, Elastic, NewRelic, Sysdig, or any other DB, you can use them in Iter8 experiments using Iter8's Custom metrics feature.

{% include ".icons/material/call-split.svg" %}

Traffic Engineering

Iter8 experiments co-exist with ingress or service mesh resources in your cluster and can leverage any features provided by them. Run experiments with traffic mirroring, traffic split based on percentage, headers, or cookies, and session affinity.

{% include ".icons/material/transit-connection-variant.svg" %}

CI/CD/GitOps

Experiments can send HTTP notifications. They can also create git pull requests. Embed them within any CI/CD/GitOps process for promoting new app versions.

{% include ".icons/material/cog-transfer-outline.svg" %}

AI-driven & Statistically Robust

Iter8 uses statistically rigorous Bayesian analysis for version assessment and multi-armed bandit algorithms for progressive traffic shifting.
