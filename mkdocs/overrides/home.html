Maximize release velocity. Optimize business value. Protect end-user experience.

{% include ".icons/material/github.svg" %} Open Source

Iter8 is 100% open source and community-driven. Available under the Apache 2 License on GitHub.

{% include ".icons/material/check-all.svg" %} SLOs

Iter8 experiments can validate your app based on the service level objectives (SLOs) specified in the experiment.

{% include ".icons/material/atom.svg" %} Load testing

Iter8 experiments can generate requests for HTTP and gRPC services, collect built-in latency and error related metrics and validate SLOs.

{% include ".icons/material/rhombus-split.svg" %} A/B(/n) testing

Grow your business with every release. Iter8 experiments can compare multiple versions based on business value and identify a winner.

{% include ".icons/material/router.svg" %} Traffic engineering

Combine Iter8 with your service mesh, ingress, or networking technology for mirroring, user segmentation, session affinity, and gradual rollout.

{% include ".icons/material/powershell.svg" %} Simple to use

Define experiments using a simple YAML file. Run them locally, in a container, inside Kubernetes, or inside your CI/CD/GitOps pipeline.

{% include ".icons/material/apps.svg" %} Awesome integrations

Experiment with any app, serverless, or ML deployment. Use metrics from any provider like Prometheus, New Relic, Sysdig, and Elastic.

{% include ".icons/material/webhook.svg" %} Eventing

Iter8 experiments can automatically create GitHub pull requests, trigger GitHub Action workflows, and send notifications to webhooks and slack.

{% include ".icons/material/chart-bar.svg" %} Flexible reporting

Iter8 can report results of the experiment in plain text or in any custom format specified by a Go template.
